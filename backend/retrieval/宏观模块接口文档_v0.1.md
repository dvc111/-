# 融合LLM Agent与图神经推理的KGQA系统 —— 模块接口文档 v0.1

---

## 0. 全局约定

| 项 | 说明 |
| --- | --- |
| 实体 ID | `entity_id`，字符串，如 `"Q1024"` |
| 关系 ID | `relation_id`，字符串，如 `"R017"` |
| 超关系 ID | `hyper_relation_id`，字符串，如 `"HR03"` |
| 三元组 | `[head_id, relation_id, tail_id]` 三元数组 |
| 问题 ID | `question_id`，每次问答唯一标识，贯穿全流程，便于调试和日志追踪 |
| 分数 | 统一为 `float`，范围 `[0, 1]` |
| 版本号 | 每个接口 JSON 顶层带 `"schema_version": "0.1"`，后续改动升版本号，避免联调时互相猜字段 |

---

## 1. Agent + 宏观检索模块 → GNN推理模块

**接口名：`macro_subgraph`**

（由 Agent 实体链接、超关系语义评分以及 Retrieval 宏观子图裁剪共同产出）

```json
{
  "schema_version": "0.1",
  "question_id": "q_0001",
  "question_text": "阿司匹林和布洛芬同时服用会有什么风险？",

  "entity_linking": [
    {
      "mention": "阿司匹林",
      "entity_id": "Q1024",
      "confidence": 0.97
    },
    {
      "mention": "布洛芬",
      "entity_id": "Q1088",
      "confidence": 0.95
    }
  ],

  "topic_entities": [
    "Q1024",
    "Q1088"
  ],

  "selected_hyper_relations": [
    {
      "relation_id": "HR03",
      "label": "药物相互作用",
      "score": 0.92
    },
    {
      "relation_id": "HR07",
      "label": "禁忌症",
      "score": 0.81
    }
  ],

  "macro_subgraph": {
    "nodes": [
      {
        "entity_id": "Q1024",
        "label": "阿司匹林",
        "is_topic": true,
        "initial_importance": 0.96
      },
      {
        "entity_id": "Q1088",
        "label": "布洛芬",
        "is_topic": true,
        "initial_importance": 0.94
      },
      {
        "entity_id": "Q2031",
        "label": "胃肠道出血",
        "is_topic": false,
        "initial_importance": 0.72
      }
    ],

    "triples": [
      ["Q1024", "R017", "Q2031"],
      ["Q1088", "R017", "Q2031"]
    ]
  },

  "max_hops": 3
}
```

### 字段说明

| 字段 | 说明 |
| --- | --- |
| `entity_linking` | Agent 完成实体识别及实体消歧后的结果，作为宏观检索的起始节点 |
| `topic_entities` | 实体链接后的主题实体集合，作为宏观检索和 GNN 推理的输入 |
| `selected_hyper_relations` | Agent 依据问题语义评分得到的超关系集合，宏观检索依据高分超关系约束图遍历方向，实现知识领域快速定位 |
| `macro_subgraph.nodes` | 宏观子图中的实体节点 |
| `initial_importance` | 节点初始重要性得分，由实体链接置信度、超关系匹配程度及节点中心性综合计算，为 GNN 节点重要性分析提供初始化特征 |
| `macro_subgraph.triples` | 宏观子图裁剪后的候选三元组集合，GNN 模块直接基于该子图进行图表示学习，无需重新访问完整知识图谱 |
| `max_hops` | 宏观检索过程中允许的最大搜索深度 |

## 2. GNN节点重要性分析 → 候选答案筛选

**接口名：`gnn_reasoning_output`**

（由 GNN 节点重要性计算及候选答案筛选模块产出）

```json
{
  "schema_version": "0.1",
  "question_id": "q_0001",

  "node_importance": [
    {
      "entity_id": "Q2031",
      "importance": 0.91
    },
    {
      "entity_id": "Q1088",
      "importance": 0.74
    },
    {
      "entity_id": "Q1024",
      "importance": 0.68
    }
  ],

  "candidate_answers": [
    {
      "entity_id": "Q2031",
      "label": "胃肠道出血",
      "prob": 0.76
    },
    {
      "entity_id": "Q2099",
      "label": "肾功能损伤",
      "prob": 0.41
    }
  ]
}
```

### 字段说明

| 字段 | 说明 |
| --- | --- |
| `node_importance` | 图神经网络完成消息传播后得到的节点重要性得分，用于衡量各实体与当前问题的相关程度 |
| `candidate_answers` | 依据节点重要性及图结构传播结果，对候选实体进行评分排序，输出 Top-K 候选答案，为后续答案生成提供候选实体集合 |
| `prob` | 候选答案最终评分，可用于排序及结果展示 |

## 3. 前端展示 / 调度层需要的最小字段

前端无需展示图计算内部细节，仅需读取以下字段即可完成知识定位及候选答案展示。

```json
{
  "question_id": "q_0001",
  "question_text": "阿司匹林和布洛芬同时服用会有什么风险？",

  "entity_linking": [
    "阿司匹林",
    "布洛芬"
  ],

  "macro_hyper_relations": [
    "药物相互作用",
    "禁忌症"
  ],

  "node_importance": [
    {
      "entity": "胃肠道出血",
      "score": 0.91
    },
    {
      "entity": "肾功能损伤",
      "score": 0.74
    }
  ],

  "top_candidates": [
    "胃肠道出血",
    "肾功能损伤"
  ]
}
```
